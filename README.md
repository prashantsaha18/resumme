# AI Resume Builder — Machine Learning + Streamlit

An end-to-end **AI Resume Builder** that uses **traditional Machine Learning** (no LLMs)
to classify resumes, predict ATS scores, match resumes against job descriptions,
recommend skills, rank multiple resumes, and generate polished ATS-friendly PDF resumes.

---

## Features

| Module | ML Technique |
|---|---|
| Resume Classification | Logistic Regression / Random Forest / SVM / Naive Bayes (best selected by F1) |
| ATS Score Prediction | Linear Regression / Random Forest Regressor / Gradient Boosting Regressor (best selected by R²) |
| Resume ↔ Job Description Matching | TF-IDF + Cosine Similarity |
| Skill Recommendation | Rule-based mapping + spaCy noun-phrase extraction (optional) |
| Resume Ranking | Composite scoring across ATS score, job match %, skills, experience |
| Resume Builder | Multi-section forms → ReportLab-generated ATS-friendly PDF (3 templates) |

## Local Setup

### 1. Clone and install dependencies

```bash
git clone <your-repo-url>
cd resume_builder_ml
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

NLTK corpora (`stopwords`, `punkt`, `wordnet`) are downloaded automatically on first run —
no manual step needed.

**Optional: spaCy for enhanced keyword suggestions.** The Job Matching page can surface
extra multi-word keyword suggestions using spaCy noun-phrase extraction. This is fully
optional — the app works correctly without it (TF-IDF matching, ATS scoring,
classification, etc. are unaffected). spaCy is *not* included in `requirements.txt` by
default because its compiled wheels aren't reliably available for every Python version
platforms like Streamlit Community Cloud may use (see Troubleshooting below). To enable
it locally:

```bash
pip install spacy
python -m spacy download en_core_web_sm
```

### 2. Generate the training dataset

```bash
python dataset/generate_dataset.py
```

> **About the dataset:** Public resume datasets (e.g. Kaggle's "Resume Dataset") require
> external downloads that aren't guaranteed to be available in every environment. This
> project ships a synthetic dataset generator that produces the *same schema* real
> datasets use (`Resume`, `Category` for classification; `Resume`, `JobDescription`,
> `ATS_Score` for regression). **To train on real-world data instead**, download a
> Kaggle resume dataset and save it as `dataset/resume_dataset.csv` /
> `dataset/ats_dataset.csv` with matching column names — `train.py` requires no other
> changes.

### 3. Train the models

```bash
python train.py
```

This trains and compares all candidate models for both tasks, prints metrics to the
console, and saves the winners to `models/`.

### 4. Run the app

```bash
streamlit run app.py
```

Open the URL Streamlit prints (typically `http://localhost:8501`).

**Self-healing models:** if `models/*.pkl` are missing, or fail to load (e.g. they were
pickled with a different numpy/scikit-learn version than what's installed), the app
automatically regenerates the dataset and retrains all models in the current environment
on first load — no manual `train.py` step required, though it's still available and
recommended for local development/CI. This is what makes deployment resilient to
whatever package versions a host like Streamlit Cloud ends up installing.

## Deployment on Streamlit Community Cloud

1. Push this repository to GitHub (see `.gitignore` — by default, generated
   `models/*.pkl` and `dataset/*.csv` files are **not** excluded, so you can commit
   them for a faster first deploy; regenerate anytime with `generate_dataset.py` +
   `train.py`).
2. Go to [share.streamlit.io](https://share.streamlit.io), connect your GitHub account,
   and select this repository.
3. Set the main file path to `app.py`.
4. Streamlit Cloud will install everything in `requirements.txt` automatically.
5. That's it — no manual training step is required. If the committed models in
   `models/` load successfully, the app uses them immediately. If they're missing or
   incompatible with whatever package versions got installed, the app automatically
   regenerates the dataset and retrains on first load (shown via a spinner,
   "First-time setup: training ML models..."). Subsequent loads reuse the cached models
   via Streamlit's `@st.cache_resource`.

### Troubleshooting: dependency install fails on Streamlit Cloud

Streamlit Community Cloud does not reliably honor `runtime.txt` for pinning a Python
version — it may build your app against a newer Python than you expect regardless of
that file (this is a known platform behavior, not specific to this project). Exact
version pins (`package==1.2.3`) are the most common cause of install failures here,
because the pinned version may not have a compiled wheel for whatever Python version
Cloud happens to assign.

**This is why `requirements.txt` uses minimum-version pins (`package>=1.2`) instead of
exact pins** — this lets `pip`/`uv` resolve whatever compatible release actually has a
wheel for the assigned Python version, rather than failing outright. Every dependency
in this project (pandas, numpy, scikit-learn, reportlab, etc.) is used only through
long-stable, version-agnostic APIs, so newer releases resolve cleanly.

If you still see an install error like *"has no wheels with a matching Python ABI tag"*
or *"No matching distribution found"* for a specific package:

- Check the deploy logs for which Python version was actually used (e.g. "Using Python
  3.14.x environment").
- Loosen or remove the version constraint on the offending package in `requirements.txt`.
- Or set the desired Python version explicitly via the **Advanced settings** dialog when
  deploying (not `runtime.txt`) — see [Streamlit's docs on upgrading Python version](https://docs.streamlit.io/deploy/streamlit-community-cloud/manage-your-app/upgrade-python).
- This project intentionally ships **without spaCy** in `requirements.txt` — spaCy's
  compiled wheels lag behind the newest Python releases and were the direct cause of a
  previous deploy failure. The app degrades gracefully without it (see the note in
  Local Setup above).

## Evaluation Metrics

- **Classification:** Accuracy, Precision, Recall, F1 Score, Confusion Matrix
  (all viewable on the **Model Performance** page).
- **Regression (ATS Score):** MAE, RMSE, R² Score (also on **Model Performance**).

---

## Disclaimer

ATS scores and job-match percentages are **statistical estimates** from models trained
on a bounded dataset — they are directional guidance, not a guarantee of how any real
company's Applicant Tracking System will score a specific resume.
